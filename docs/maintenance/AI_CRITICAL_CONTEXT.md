## 1. ゼロ幅文字 (ZWC) ステガノグラフィの不可侵条項
- **場所**: `src/streaming.js` (`injectZwcInvisibleMetadata`)
- **事実**: 本システムは、Gemini APIのストリーミング応答の中に、人間には見えない「ゼロ幅文字（`\u200B`, `\u200C`, `\u200D`）」を用いて、JSON形式のメタデータ（実行したツールの結果等）を密輸しています。
- **絶対の掟**: **これを「表示の不具合（ゴミデータ）」や「不正な文字」と誤認して、トリム（`.trim()`）やサニタイズ（削除）を行ってはいけません。**
- **理由**: Telegram等のチャットUIを汚さずに、OpenClaw（Gateway）からGemini CLIへツール履歴を非同期伝達するための最新かつ唯一の安定した手法（SSoT 3.1 Architecture）だからです。削除するとAIは過去の記憶（ツール結果）を失います。

## 2. 履歴の強制注入処理の禁止 (JSONL Write)
- **事実**: かつて、OpenClaw側の履歴ファイル（`.jsonl` ファイル）に対して正規表現を用いて強引に `tool_use` ブロックを書き込む手法（SSoT 2.0）を採用していましたが、ストリームフォーマットの競合によりシステムが崩壊しました。
- **絶対の掟**: **履歴ファイル（`.jsonl` 等）への直接書き込みによる履歴同期手法を二度と提案・実装しないでください。**
- **現在の正解**: 履歴はすべて中継アダプタ側（`src/runner-pool.js` 経由の IPC）のメモリ内 `resumedSessionData` ベースで解決する設計となっています。

## 3. プロキシおよびサンドボックス解除の背景 (YOLO Mode)
- **場所**: `~/.gemini/settings.json` および `interactive-setup.js`
- **事実**: 本システムは、Gemini CLIをフル自律モード（`--yolo`）かつ、すべてのディレクトリへのアクセスを許可したサンドボックス無効状態（`tools.sandbox: false`）で動作させます。
- **理由**: 自律エージェント（OpenClaw）がOSコマンドや任意のファイル操作をプロンプトなしで連続実行するための**意図的な仕様**です。「セキュリティリスクがあるから」といって勝手にサンドボックスを有効化（`true`）すると、エージェントが身動きできなくなります。

## 4. パス解決のフォールバック (npmバイナリ起動)
- **事実**: アダプタが `mcp-server.mjs` やGemini CLIの設定を読み込む際、インストールディレクトリは環境に依存します。
- **絶対の掟**: パスの解決には必ず環境変数 `GEMINI_CLI_HOME` を起点にするか、実行時に動的にパスを探索する構造を維持してください。`~/.openclaw` や `/root` といったハードコードされた絶対パスを信用しないでください。（現在リファクタリング中の技術負債でもあります）。
